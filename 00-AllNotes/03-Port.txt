1. INTRODUCTION TO PORTS
================================================================================

What is a Port?
---------------
A port is a communication endpoint in computer networking. It's like a door number in a building - it tells the computer which application or service should receive incoming data.

Why Do We Need Ports?
- **Multiple Services**: One computer can run many services (web server, database, etc.)
- **Process Identification**: Operating system knows which process gets which data
- **Security**: Firewalls can control access based on ports
- **Organization**: Structured way to manage network traffic

Port Analogy:
Think of your computer as a hotel:
- IP address = Hotel address
- Port = Room number
- When data arrives at the hotel (IP), it goes to the correct room (port)

2. TCP VS UDP - TRANSPORT PROTOCOLS
================================================================================

TCP (Transmission Control Protocol):
-----------------------------------
- **Connection-oriented**: Establishes connection before sending data
- **Reliable**: Guarantees delivery, order, and error checking
- **Slower**: More overhead due to reliability features
- **Use cases**: Web browsing, email, file transfer

Characteristics:
- Three-way handshake (SYN, SYN-ACK, ACK)
- Flow control and congestion control
- Retransmission of lost packets
- Suitable for applications where data integrity is crucial

UDP (User Datagram Protocol):
-----------------------------
- **Connectionless**: No connection setup required
- **Unreliable**: No guarantee of delivery or order
- **Faster**: Less overhead, lower latency
- **Use cases**: Video streaming, gaming, DNS, VoIP

Characteristics:
- Fire-and-forget approach
- No retransmission or ordering
- Smaller packet headers
- Better for real-time applications where speed matters more than perfection

TCP vs UDP Comparison:

| Feature          | TCP                          | UDP                          |
|------------------|------------------------------|------------------------------|
| Connection      | Connection-oriented          | Connectionless              |
| Reliability     | Reliable                     | Unreliable                   |
| Speed           | Slower                       | Faster                       |
| Overhead        | High                         | Low                          |
| Use Cases       | HTTP, FTP, SMTP             | DNS, DHCP, Video streaming   |
| Ordering        | Maintains order              | No ordering guarantee        |

3. PORT RANGES AND TYPES
================================================================================

Port Number Range:
------------------
Ports are 16-bit numbers, ranging from 0 to 65,535 (2^16 - 1).

Port Categories:

1. **Well-Known Ports (0-1023)**
   - Assigned by IANA (Internet Assigned Numbers Authority)
   - Standard services and protocols
   - Require administrator/root privileges on most systems
   - Examples: 80 (HTTP), 443 (HTTPS), 22 (SSH)

2. **Registered Ports (1024-49151)**
   - Registered with IANA for specific services
   - Can be used by applications without special privileges
   - Examples: 3306 (MySQL), 5432 (PostgreSQL), 27017 (MongoDB)

3. **Dynamic/Private Ports (49152-65535)**
   - Also called ephemeral ports
   - Used for temporary connections
   - Assigned dynamically by operating system
   - Used by client applications when connecting to servers

Port Registration:
- Organizations can register ports with IANA
- Prevents conflicts between different applications
- Official list available at: https://www.iana.org/assignments/service-names-port-numbers/

4. COMMON PORTS AND SERVICES
================================================================================

Web Services:
- **80**: HTTP (HyperText Transfer Protocol)
- **443**: HTTPS (HTTP Secure)
- **8080**: HTTP Alternative (common for development)

Remote Access:
- **22**: SSH (Secure Shell)

- **995**: POP3S (POP3 over SSL)

Database Services:
- **3306**: MySQL
- **5432**: PostgreSQL
- **27017**: MongoDB
- **6379**: Redis
- **9200**: Elasticsearch
- **9042**: Cassandra

File Transfer:
- **20/21**: FTP (File Transfer Protocol)

5. HOW PORTS WORK IN NETWORKING
================================================================================

Socket Concept:
---------------
A socket is the combination of IP address and port number.
- **Socket = IP Address + Port Number**
- Example: 192.168.1.100:80

How Data Flows:
1. **Client** wants to connect to a service
2. **Client** creates a socket with random ephemeral port
3. **Client** sends request to server's IP + well-known port
4. **Server** receives request on its port
5. **Server** processes request and sends response
6. **Communication** happens through the established connection

Port Binding:
- Applications "bind" to specific ports
- Only one application can bind to a port at a time
- Binding requires the port to be available

Example: Web Server
```javascript
const express = require('express');
const app = express();

app.listen(3000, () => {
  console.log('Server listening on port 3000');
});
// Server binds to port 3000
```

6. PORT FORWARDING AND NAT
================================================================================

What is Port Forwarding?
------------------------
Port forwarding allows external traffic to reach devices inside a private network.

Why Needed:
- Home routers use private IP addresses (192.168.x.x)
- External devices can't directly access internal devices
- Port forwarding maps external ports to internal IP:port combinations

Types of Port Forwarding:
1. **Static Port Forwarding**: Permanent mapping
2. **Dynamic Port Forwarding**: SSH tunneling
3. **UPnP Port Forwarding**: Automatic by applications

Example Router Configuration:
- External Port: 80 → Internal IP: 192.168.1.100:3000
- When someone accesses router's public IP on port 80
- Traffic gets forwarded to internal server on port 3000

NAT (Network Address Translation):
---------------------------------
- Translates private IP addresses to public IP addresses
- Allows multiple devices to share one public IP
- Types: Static NAT, Dynamic NAT, PAT (Port Address Translation)

Security Considerations:
- Port forwarding can expose internal services
- Use firewall rules to restrict access
- Consider VPN for secure remote access

7. FIREWALLS AND PORT SECURITY
================================================================================

What is a Firewall?
-------------------
A network security system that monitors and controls incoming and outgoing network traffic based on predetermined security rules.

Port-based Security:
- **Allow**: Permit traffic on specific ports
- **Deny**: Block traffic on specific ports
- **Drop**: Silently discard packets

Common Firewall Rules:
```bash
# Allow SSH (port 22) from specific IP
iptables -A INPUT -p tcp -s 192.168.1.100 --dport 22 -j ACCEPT

# Block all incoming traffic except allowed ports
iptables -P INPUT DROP
iptables -A INPUT -p tcp --dport 80 -j ACCEPT
iptables -A INPUT -p tcp --dport 443 -j ACCEPT
```

Security Best Practices:
- **Principle of Least Privilege**: Open only necessary ports
- **Default Deny**: Block everything, allow specific
- **Regular Audits**: Check open ports periodically
- **Use VPN**: For remote access instead of exposing ports
- **Intrusion Detection**: Monitor suspicious activity

Port Scanning:
- Attackers scan for open ports
- Tools like nmap can discover services
- Keep services updated and use firewalls

8. SOCKET PROGRAMMING BASICS
================================================================================

What is Socket Programming?
--------------------------
Programming interface for network communication using sockets.

Client-Server Socket Flow:

**Server Side:**
1. Create socket
2. Bind to IP and port
3. Listen for connections
4. Accept connections
5. Send/receive data
6. Close connection

**Client Side:**
1. Create socket
2. Connect to server IP and port
3. Send/receive data
4. Close connection

Node.js TCP Server Example:
```javascript
const net = require('net');

const server = net.createServer((socket) => {
  console.log('Client connected');

  socket.on('data', (data) => {
    console.log('Received:', data.toString());
    socket.write('Hello from server!');
  });

  socket.on('end', () => {
    console.log('Client disconnected');
  });
});

server.listen(3000, () => {
  console.log('Server listening on port 3000');
});
```

Node.js TCP Client Example:
```javascript
const net = require('net');

const client = net.createClient({ port: 3000, host: 'localhost' });

client.on('connect', () => {
  console.log('Connected to server');
  client.write('Hello from client!');
});

client.on('data', (data) => {
  console.log('Received:', data.toString());
  client.end();
});
```

9. TOOLS AND COMMANDS FOR WORKING WITH PORTS
================================================================================

Network Diagnostic Tools:

**netstat** (Network Statistics):
```bash
# Show all listening ports
netstat -tlnp

# Show all connections
netstat -tunap

# Show specific port
netstat -tlnp | grep :80
```

**ss** (Socket Statistics - modern replacement for netstat):
```bash
# Show listening sockets
ss -tlnp

# Show all sockets
ss -tunap
```

**lsof** (List Open Files):
```bash
# Show processes using port 80
lsof -i :80

# Show all network files
lsof -i
```

**nmap** (Network Mapper):
```bash
# Scan local machine ports
nmap localhost

# Scan specific ports
nmap -p 80,443 localhost

# Service version detection
nmap -sV localhost
```

**telnet/netcat** (Testing connections):
```bash
# Test if port is open
telnet localhost 80

# Using netcat
nc -zv localhost 80
```

**curl** (Testing HTTP services):
```bash
# Test HTTP endpoint
curl http://localhost:3000

# Test with headers
curl -I http://localhost:3000
```

10. TROUBLESHOOTING PORT ISSUES
================================================================================

Common Port Problems:

1. **Port Already in Use**
   - Another application is using the port
   - Solution: Kill process or use different port
   ```bash
   # Find process using port 3000
   lsof -ti:3000 | xargs kill -9
   ```

2. **Permission Denied (Ports < 1024)**
   - Need root/admin privileges for well-known ports
   - Solution: Use sudo or choose port > 1024

3. **Connection Refused**
   - No service listening on that port
   - Firewall blocking connection
   - Wrong IP address or port

4. **Address Already in Use**
   - Previous connection not properly closed
   - TIME_WAIT state in TCP
   - Solution: Wait or use SO_REUSEADDR option

5. **Port Forwarding Issues**
   - Router not configured correctly
   - ISP blocking ports
   - Double NAT (router behind router)

Debugging Steps:
1. Check if service is running: `ps aux | grep service`
2. Check listening ports: `netstat -tlnp`
3. Test local connection: `curl localhost:port`
4. Check firewall: `iptables -L`
5. Test from another machine
6. Check router port forwarding

Development Tips:
- Use ports > 3000 for development
- Document which ports your application uses
- Use environment variables for port configuration
- Handle port conflicts gracefully
